## What is Software Engineering?

  When I enrolled into my ICS 314 Software Engineering class, I definitely thought the class was simply about web application development. It excited me because it was something new that I've never learned before and the thought of building web apps always seemed interesting to me. I mean, who wouldn't enjoy making their own websites? I didn't know that "software engineering" came with substantially more concepts though, such as development environments and group management practices. This class was more than just about coding a website and I could especially see that toward the last few weeks of the semester. 
  
   At the beginning of this class, our professor gave a presentation introducing what the class was going to be like. One of the slides presented our final group project where we would develop our own application at the end of the course. At that point, I should have known that this class style would be different than other classes I've taken. I soon learned that this class was all about how to manage ones professional profile and learning skills that would be useful to programmers for their career. We learned concepts such as open source software development and agile project management, which I would most likely see in the future. So there was more to this class than just building a website.  

## The fundamental thing of modern software

  Believe it or not, I didn't know much about open source software before this class. I have heard the concept, but it was something that I wasn't properly introduced to before. In fact GitHub, probably one of the forerunners of open source software, was only familiar to me because I would Google code examples and it was one of the top searches. I honestly had no idea how to use it. A few weeks into the class and after a little practice, we always used GitHub to host our projects. It was convenient, and also easy to share and duplicate code. And that's pretty much what open source software is! Being able to easily access code online in order to learn more and improve code. It's a big part of modern programming today and I'm actually a little surprised that I haven't learned to use this practice earlier.
  
  GitHub software was very useful for this class. Our class duplicated a lot of our professors code and built off of them to improve them and learn different frameworks. They provided good examples to reference for when I got stuck on assignments and for learning how certain things should work. It was especially useful when learning syntax I wasn't familiar with. 
  
## The agile way for teamwork

  Another valuable concept we learned in this class was agile project management. If I'm being honest, I totally didn't know what this concept meant when first writing this essay, but that's because we focused on a more specific form of agile project management called issue driven project management. Now agile project management is all about how to work with a team. It's a strategy that splits the work into individual tasks for each member and allows for planning to be constantly adapted. This lets members focus on their work individually and makes tasks seem more approachable in smaller steps. It also allows for effective improvements because plans can always be reassessed depending on the projects progress. 
  
  Issue driven project management is an agile form that focuses on tasks being created as issues, where you can move forward in a project based on what needs to be accomplished. We used issue driven project management in our final group project through the help of GitHub. There's a feature that supports this style of management on GitHub, where we can create issues for each milestone of our project and assign them to different members to complete. Each issue became a branch of our master program and we would make specific changes only in the issue we were assigned to. If we couldn't complete an issue by the end date of a milestone, we would move it onto the next one. This style of doing projects was really helpful for our class because we were allowed to work more independently and didn't have to rely on meeting up too frequently for our tasks. I think this is was very organized way of group management and is very well implemented on GitHub. 

## More than just web application

  After my software engineering class, I learned that there is much more to software development than I thought. As probably one of the biggest parts of modern computer science, it's concepts are very complex and fleshed out from many years of development. The concepts I learned were very valuable in building my knowledge of the practicality of programming and I look forward to applying it in the future. Open source will be a great resource that I'll probably use forever and GitHub will be a great tool that I'll always rely on. I can also see myself using issue driven project management as part of my company later on or whenever I work in a team. I could even incorporate it into future group projects for classes if I ever have any. Projects require multiple steps after all, so splitting them this way makes it easier for setting goals and outlining my progress. I think software development is a good direction for me to head into. It was interesting creating my own applications, but there is a lot more I can learn about software development and I can't wait to do more.
